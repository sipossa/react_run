@YoungkeunAhn
setForm(form => ({ ...form, [name]: value })); 에서 form은 prevState를 의미합니다.

useState 훅의 setState함수는 비동기라
실행 한 후 state값을 조회할 때 최신 state값의 반환을 보장하지 않습니다.

그렇기 때문에 이전 상태값이 업데이트될 상태값에 영향을 미치는 경우에
setState함수는 prevState를 인자로 받는 콜백을 받아 처리하는것을 권장합니다.

@YoungkeunAhn
useCallback함수의 deps에 빈 배열을 전달했기 때문에 리렌더링 될 때마다 호출하게 됩니다. 이 때 dotdotgod님의 답변처럼 최신 state값을 보장하지 않기 때문에 현재 입력하고 있는 input을 제외한 나머지 값은 갱신이 되지 않은 채로 호출됩니다. 게시된 코드처럼 함수형 업데이트를 하지 않는다면 deps에 form을 추가해야 해당 버그가 사라집니다.

const onChange = useCallback((e) => {
	const {name, value} = e.target;
	setForm({...form, [name]: value});
}, [form]);
